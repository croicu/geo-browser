- **Method** вЂ” Python calls JavaScript. Python initiates, JS handles and returns a response. Analogous to an RPC call.
- **Event** вЂ” JavaScript calls Python. JS initiates, Python handles and optionally returns a value (e.g. a cancel flag).

Both directions use the same wire format and the same `window.__geo_dispatch` entry point.

## API Response Contract

Every API response payload must carry three fields:

| Field | Type | Description |
|-------|------|-------------|
| `error` | `number` | `0` = success (`OK`); non-zero = caller-defined error code |
| `errorDescription` | `string \| null` | Human-readable detail; `null` when `error === OK` |
| payload fields | varies | Domain data (e.g. `bbox`); may be absent or `null` when `error !== OK` |

**Rules:**
- Error codes are part of the API contract and are declared by the Python caller in `api.py`.
- The TypeScript callee must always check `error` before using payload fields.
- Branch on the numeric code, not `errorDescription` вЂ” that field is for logging only.
- `OK = 0` is the only universal constant; all other codes are API-specific.

## Wire Protocol

All messages are JSON objects posted via `window.chrome.webview.postMessage` (JSв†’Python) or injected via `window.__geo_dispatch(msg)` (Pythonв†’JS).

### New call or event fire

```jsonc
{ "id": "<method-or-event-id>", "data": { ... } }
```

### New call or event fire expecting a response

```jsonc
{ "id": "<method-or-event-id>", "requestId": "<opaque-string>", "data": { ... } }
```

### Response (either direction)

```jsonc
{ "requestId": "<opaque-string>", "data": { ... } }
```

> **Routing rule:** a message with `id` is a new call/event. A message with only `requestId` (no `id`) is a response to a previous call. The `requestId` is an opaque string generated by the caller; it is echoed back unchanged in the response.

## `startup.js` вЂ” the bridge (already implemented)

`startup.js` is injected into every page by the Python host. It sets up:

```typescript
// Python в†’ JS entry point. Called by the Python host to deliver a message.
window.__geo_dispatch(msg: WireMessage): void

// JS public API
window.geo = {
  // Fire an event to Python.
  // Pass callback to receive Python's response.
  invoke(id: string, payload: unknown, callback?: (data: unknown) => void): void

  // Register a handler for a Python method call.
  // The return value of fn is sent back to Python as the response.
  subscribe(id: string, fn: (data: unknown) => unknown): void

  unsubscribe(id: string): void
}
```

`startup.js` maintains two internal tables:
- `_handlers` вЂ” JS-side method handlers registered via `subscribe`
- `_pending` вЂ” pending callbacks for event responses, keyed by `requestId`

## `api.ts` вЂ” what to implement

Mirror `src/geo_builder/api.py` on the TypeScript side. Suggested shape:

```typescript
type Cookie = number;

interface MethodDef<TIn, TOut> {
  id: string;
  // TIn  = what Python sends (deserialize from data)
  // TOut = what JS returns (serialized as the response)
}

interface EventDef<TIn, TOut> {
  id: string;
  // TIn  = what JS sends
  // TOut = what Python returns (deserialize from response data)
}

class Gateway {
  // Register a handler for a Python method call.
  // fn receives typed input; its return value is sent back to Python.
  subscribe<TIn, TOut>(def: MethodDef<TIn, TOut>, fn: (data: TIn) => TOut): void

  // Fire an event to Python.
  invoke<TIn, TOut>(def: EventDef<TIn, TOut>, data: TIn, callback?: (response: TOut) => void): void

  // Subscribe to / unsubscribe from a Python event.
  // (For events that Python fires repeatedly, e.g. catalog updates.)
  register<TIn, TOut>(def: EventDef<TIn, TOut>, fn: (data: TIn) => TOut | void): Cookie
  unregister(cookie: Cookie): void
}
```

## Ping / Pong (connection handshake)

Ping/Pong verifies the bridge is alive after each page load. Python initiates with a Ping method call; JavaScript responds with a Pong event.

**Flow:**

1. Python calls `gateway.ping()` вЂ” method call *with* a response callback.
   Wire message to JS: `{ "id": "__geo_ping__", "requestId": "<rid>", "data": { "token": "<token>" } }`

2. `api.ts` must subscribe to Ping. The handler should:
   - return the token (exercises the method-response path в†’ `startup.js` sends `{ requestId, data }` back to Python)
   - also fire a Pong event (exercises the JS-initiated event path)
   ```typescript
   gateway.subscribe(Ping, ({ token }) => {
     gateway.invoke(Pong, { token });
     return { token };           // echoed back as the method response
   });
   ```

3. Python receives both confirmations independently:
   - The method response (`{ requestId, data }`) triggers `_on_ping`, which sets `gateway.is_open = True`.
   - The Pong event (`{ id: "__geo_pong__", data }`) triggers `_on_pong`, which also sets `gateway.is_open = True`.
   The second arrival is a harmless no-op.

**`api.ts` must implement the Ping handler** вЂ” see step 2 above. The connection is open once `gateway.is_open` is `true` on the Python side.

## Adding a new method (Python в†’ JS)

**Python (`gateway.py` or caller):**
```python
@dataclass
class FocusAreaInput:
    areaId: str

# In api.py (shared):
# @dataclass class FocusAreaInput: areaId: str

gateway.define_method("__geo_focus_area__", FocusAreaInput)   # fire-and-forget
# or with response:
# gateway.define_method("__geo_focus_area__", FocusAreaInput, FocusAreaOutput)

gateway.call("__geo_focus_area__", FocusAreaInput(areaId="paris"))
```

**TypeScript (`api.ts`):**
```typescript
interface FocusAreaInput { areaId: string; }

const FocusArea: MethodDef<FocusAreaInput, void> = { id: "__geo_focus_area__" };

gateway.subscribe(FocusArea, ({ areaId }) => {
  // handle focus
});
```

## Adding a new event (JS в†’ Python)

**TypeScript (`api.ts`):**
```typescript
interface AreaSelectedInput  { areaId: string; }
interface AreaSelectedOutput { cancel: boolean; }

const AreaSelected: EventDef<AreaSelectedInput, AreaSelectedOutput> = { id: "__geo_area_selected__" };

gateway.invoke(AreaSelected, { areaId: "paris" }, (response) => {
  if (response.cancel) { /* Python cancelled the selection */ }
});
```

**Python (`gateway.py` or caller):**
```python
@dataclass
class AreaSelectedInput:
    areaId: str

@dataclass
class AreaSelectedOutput:
    cancel: bool

gateway.define_event("__geo_area_selected__", AreaSelectedInput, AreaSelectedOutput)

def on_area_selected(data: AreaSelectedInput) -> AreaSelectedOutput:
    return AreaSelectedOutput(cancel=False)

gateway.register("__geo_area_selected__", on_area_selected)
```

## GetAreaBbox (`__geo_get_area_bbox__`)

Returns the bounding box (GPS) of an area given its ID. The bbox is derived from `area.center` and `area.radiusMeters`.

**TypeScript:**
```typescript
const GetAreaBbox: EventDef<GetAreaBboxInput, GetAreaBboxOutput> = { id: "__geo_get_area_bbox__" };

gateway.invoke(GetAreaBbox, { areaId: "paris" }, ({ error, bbox }) => {
  if (error !== OK) return;
  // bbox = [west, south, east, north]
});
```

**Python:** handled by `Catalog.register_handlers(gateway)` вЂ” no manual wiring needed.
